# Parameter Fitting with PINTS

This directory contains the parameter fitting pipeline for the HPA axis model using the PINTS optimization framework.

## Overview

The `run_pints.py` script fits model parameters to experimental ACTH and cortisol data for individual participants using CMA-ES optimization. This was run on a high-performance computing (HPC) cluster due to the computational demands of the optimization process.

## Usage

```bash
python pints_fitting/run_pints.py --config parameters.json
python pints_fitting/run_pints.py --config parameters_9.json --phase_align 1
```

## Arguments

- `--config` - Configuration file name (from `model/config/base/`)
- `--phase_align` - Align model to data by phase shifting (1) or fit phase parameter t_s (0). Default: 0
- `--animation` - Generate optimization animation (1) or skip (0). Default: 0

## Configuration

Each participant has a configuration file in `model/config/base/` specifying:
- Initial parameter guesses
- Parameters to fit vs. fixed parameters
- Data range to use for fitting
- Number of simulation days

## Optimization Details

- **Algorithm:** CMA-ES (Covariance Matrix Adaptation Evolution Strategy)
- **Max iterations:** 1500
- **Error function:** Sum of squared errors between observed and simulated ACTH and cortisol
- **Data:** 1-minute resolution ACTH and cortisol time series

## Output

Results are saved to `pints_fitting/output/{participant}.{run}/`:
- `parameters.txt` - Fitted parameter values
- `fitting_plot.png` - Observed vs. fitted time series
- `error_evolution.png` - Error reduction during optimization
- `optimization.gif` - Animation of fitting process (if `--animation 1`)
- `log.txt` - Optimization log with iteration details

## HPC Execution

This fitting pipeline was executed on an HPC cluster to enable parallel fitting of multiple participants. 

## Notes

- Data files must exist in `model/data/` (generated by `data_analysis/code/data_prep.py`)
- First day of simulation is discarded to allow transient effects to settle
- Phase alignment (`--phase_align 1`) is useful when circadian phase (t_s) is not being fitted
